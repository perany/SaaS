三.开始使用

  • 1.发现问题
    • 通过“整体概览”方便管理层发现应用运营情况
    • 1.1整体概览
    • 在“整体概览”里，可以监测应用实时、近期、和历史的整体用户、整体营收情况；
    • 整体概览指标说明：
    • 1. 时间维度：以“今日、近期：近7日/近30日、历史”统计；
    • 2.“新增用户”为：所选周期范围内，安装应用后首次启用应用的用户；
    • 3.“活跃用户”为：所选周期范围内，至少启动过一次的用户；
    • 4.“注册用户”为：所选周期范围内，有注册成功行为的用户；
    • 5.“付费用户”为：所选周期范围内，有付费成功行为的用户；
    • 6.“新增用户历史平均值”＝数据对接进来至今天的新增用户／全部天数；
    • 7.“活跃用户历史平均值”＝数据对接进来至今天的活跃用户／全部天数；
    • 8.“注册用户历史平均值”＝数据对接进来至今天的注册用户／全部天数；
    • 9.“付费用户历史平均值”＝数据对接进来至今天的付费用户／全部天数；
    • 10.“累计用户”＝数据对接进来至今天的累积新增用户相加；
    • 11.“累计注册用户”＝数据对接进来至今天的注册用户相加；
    • 12.“累计付费用户”＝数据对接进来至今天的付费用户相加；
    • 13. 用户在查看用户趋势时，能清晰辨识是否有新版本上线，从而判断流量高低是否由版本上线导致；版本标记：版本号、标记时间定义为版本上线起始日；
    • 14. 用户在查看用户趋势时，能清晰辨识是否有新活动，从而判断流量高低是否由举办活动导致 ；活动标记：活动名称、活动周期，标记时间为活动起始日；
    • 15. 用户查看用户趋势时，能清晰辨识是否有波动点，从而判断流量预警；波动点标记“与上一时期上下超出50%”、公式为：［ (本统计周期数据-上统计周期数据)/上统计周期数据× 100% )；默认全部用户：不做标记；当切换到新增用户、活跃用户、注册用户、付费用户时，标记：波动点；版本、活动；
  • 2.定位问题
  • 通过“用户分析”、“行为分析-漏斗分析、使用分析”快速定位应用运营情况
    • 2.1用户分析
    • 在“用户分析”中，通过用户对比、影响度分布、渠道分布差异比、版本分布差异比定位应用运营情况；
    • 用户分析指标说明
    • 1. 用户类型：以“新增用户、活跃用户、注册用户、付费用户”类型筛选；
    • 2. 时间维度：以“T+1自定义选择”，对比周期以“过去7天/30天/60天平均值”；
    • 3. 用户对比：不同用户类型 在“所选周期内的人数”和“过去平均值”比较； 当周期选择1天时，以“小时段”显示，与之比较的为“过去时段中的平均值”； 当周期选择等于或大于2天时，以“日”显示，与之比较的为“过去天数的平均值”；
      • 3.1 用户在查看用户趋势时，能清晰辨识是否有新版本上线，从而判断流量高低是否由版本上线导致；版本标记：版本号、标记时间定义为版本上线起始日；
      • 3.2 用户在查看用户趋势时，能清晰辨识是否有新活动，从而判断流量高低是否由举办活动导致 ；活动标记：活动名称、活动周期，标记时间为活动起始日；
      • 3.3 用户查看用户趋势时，能清晰辨识是否有波动点，从而判断流量预警；波动点标记“与上一时期上下超出50%”、公式为：［ (本统计周期数据-上统计周期数据)/上统计周期数据× 100% )；默认全部用户：不做标记；当切换到新增用户、活跃用户、注册用户、付费用户时，标记：波动点；版本、活动；
    • 4. 差异度分布：所选用户类型数量在“渠道&版本”中，与该类型用户过往平均数量的差异度高中低；（*例.当新增用户在1月1日暴跌时，当天渠道差异度显示高，版本差异度显示低，则可定位问题为当天渠道内新增用户数量有明显大波动，较过去7天平均水平差异度大，可后续通过“渠道分析”页面进一步解析具体原因）；
    • 5. 渠道差异率：所选用户类型数量在各个渠道中的“周期平均人数”和“过去x天平均值”，以及两者对比的差异率（-100%-+100%）；
    • 6. 版本差异率：所选用户类型数量在各个版本中的“周期平均人数”和“过去x天平均值”，以及两者对比的差异率（-100%-+100%）；
    • 2.2行为分析
      • 2.2.1漏斗分析
      • 通过“漏斗分析”定位不同的行为事件在操作步骤中的转化情况，可以定位事件访问人数、步骤转化率、渠道分布差异比、版本分布差异比情况；
      • 漏斗分析指标说明：
      • 1. 单个事件组由多个事件步骤构成；事件&事件组可在自定义管理模块中定义（暂未开放，请联络管理员获得后台支持）；
      • 2. 时间维度：以“T+1自定义选择”，对比周期以“过去7天/30天/60天平均值”；
      • 3. 访问人数：统计每步事件在“周期内的访问人数”和“过去平均值”比较；
      • 4. 步骤转化率：统计每步事件访问人数、流失用户、单步转化率、整体转化率；
      • 5. 渠道分布差异率：默认为事件组在各个渠道中的“ 周期访问人数”&“过去平均值”，以及两者对比的差异率（-100%-+100%）；当选择单步事件时，统计各个渠道中的“ 周期平均访问人数”&“过去平均值”，以及两者对比的差异率（-100%-+100%）；
      • 6. 版本分布差异率：默认为事件组在各个版本中的“ 周期访问人数”&“过去平均值”，以及两者对比的差异率（-100%-+100%）；当选择单步事件时，统计各个版本中的“ 周期平均访问人数”&“过去平均值”，以及两者对比的差异率（-100%-+100%）；
      • 2.2.2使用分析
      • 通过“使用分析”定位启用次数、使用时长的影响度分布、渠道差异比、版本差异比情况；
      • 使用分析指标说明：
      • 1. 使用分析由启用次数、使用时长两类统计；
      • 2. 时间维度：以“T+1自定义选择”，对比周期以“过去7天/30天/60天平均值”；
      • 3. 启用次数：“周期内的启用次数”和“过去7天/30天/60天平均启用次数”比较； 当周期选择1天时，以“小时段”显示，与之比较的为“过去天数中的时段平均值”； 当周期选择等于或大于2天时，以“日”显示，与之比较的为“过去天数的平均值”；
      • 4. 平均使用时长：“周期内的平均使用时长”和“过去7天/30天/60天平均使用时长”比较；平均使用时长＝使用总时长/启用次数；当周期选择1天时，以“小时段”显示，与之比较的为“过去天数中的时段平均值”； 当周期选择等于或大于2天时，以“日”显示，与之比较的为“过去天数的平均值”；
      • 5. 差异度分布：“启用次数&平均使用时长”数值在“渠道&版本”中，与“启用次数&平均使用时长”过往平均数值的差异度高中低；（*例.当使用时长在1月1日突涨时，当天版本差异度显示高，渠道差异度显示低，则可定位问题为当天版本维度下使用时长数值有明显大波动，较过去30天平均水平差异度大，可后续通过“功能分析”页面进一步解析具体原因）；
      • 6. 渠道差异率：“启用次数&平均使用时长”在各个渠道中的“周期平均数值”和“过去平均值”，以及两者对比的差异率（-100%-+100%）；
      • 7. 版本差异率：“启用次数&平均使用时长”在各个版本中的“周期平均数值”和“过去平均值”以及两者对比的差异率（-100%-+100%）；
  • 3.解决问题
  • 通过“渠道分析”、“活动分析”、“功能分析”、“内容分析”、“画像分析”模块实现追溯并帮助其渠道部、市场部、产品设计部、内容运营部解决应用运营问题
    • 3.1渠道分析
    • 在“渠道分析”里，为渠道部解决问题提供渠道用户比、渠道价值分布、渠道新增用户成本、渠道活跃用户成本、渠道留存用户成本、渠道付费用户成本数据；
    • 渠道分析指标说明：
    • 1. 时间维度：以“T+1自定义选择”；渠道由自定义管理模块设定；
    • 2. 渠道用户比：周期内单个渠道的单类用户占全部渠道的单类用户比；
    • 3. 渠道新增用户成本：渠道新增用户成本＝投放金额/新增用户；
    • 4. 渠道活跃用户成本：渠道活跃用户成本＝投放金额/新增用户；
    • 5. 渠道留存用户成本分两步处理：
      • 5.1 留存用户数：日留存>2天，周留存>2周，月留存>2月；
        • 日留存用户：（第一天新增用户在第二天登录过的人数），以日存储次日留存用户；
        • 周留存用户：（第一周新增用户在第二周登录过的人数）；
        • 月留存用户：（第一月新增用户在第二月登录过的人数）；
      • 5.2 留存用户成本：日留存用户成本＝每日投放金额/日留存用户；
        • 周留存用户成本：周投放金额/周留存用户；
        • 月留存用户成本：月投放金额/月留存用户；
    • 6. 付费数：日付费＝1天，周付费>1周，月付费>1月；
    • 付费用户：周期范围内产生过至少一次成功支付行为的用户；
    • 日付费用户成本＝每日投放金额/日付费用户；
    • 周付费用户成本：周投放金额/周付费用户；
    • 月付费用户成本：月投放金额/月付费用户；
    • 3.2活动分析
    • 在“活动分析”里，为市场部解决问题提供活动价值分布、活动新增用户成本、活动活跃用户成本、活动留存用户成本、活动付费用户成本分析；
    • 活动分析指标说明：
    • 1. 时间维度：以“T+1自定义选择”活动由自定义管理模块设定；
    • 2. 活动新增用户成本：活动新增用户成本＝投放金额/新增用户；
    • 3. 活动活跃用户成本：活动活跃用户成本＝投放金额/新增用户；
    • 4. 活动留存用户成本分两步处理：
      • 4.1 留存用户数：日留存>2天，周留存>2周，月留存>2月；
        • 日留存用户：（第一天新增用户在第二天登录过的人数），以日存储次日留存用户；
        • 周留存用户：（第一周新增用户在第二周登录过的人数）；
        • 月留存用户：（第一月新增用户在第二月登录过的人数）；
      • 4.2 留存用户成本：日留存用户成本＝每日投放金额/日留存用户；
        • 周留存用户成本：周投放金额/周留存用户；
        • 月留存用户成本：月投放金额/月留存用户；
    • 5. 付费数：日付费＝1天，周付费>1周，月付费>1月；
    • 付费用户：周期范围内产生过至少一次成功支付行为的用户；
    • 日付费用户成本＝每日投放金额/日付费用户；
    • 周付费用户成本：周投放金额/周付费用户；
    • 月付费用户成本：月投放金额/月付费用户；
    • 3.3功能分析
    • 在“功能分析”里，分为版本分析，为产品部解决问题提供“版本新增用户”、“版本活跃用户”、“版本注册用户”、“版本付费用户”、“版本留存用户”，监测不同版本下的用户喜好度；
    • 版本分析指标说明：
    • 1.取值条件：版本数据源区分Android或IOS操作系统；
    • 2.取数范围： 选择时间控件在 IOS操作系统下，可选起始日期为： 2017年7月6日，未到的日期不可选；在Android操作系统下，可选起始日期为： 2017年5月16日， 未到的日期不可选； 选择时间默认为： T+30，例：今日2017-08-31、选择日期为2017-08-01至2017-08-30；当自定义选择时间等于1天时，x轴显示00:00-23:00时间数据，当自定义选择时间大于1天时，x轴显示年-月-日的天数据；当自定义选择后无数据时返回时，显示“暂无数据，请重新选择时间”；当自定义选择后已过的日期中无数据时，显示“0人”；
    • 3.版本对比：默认显示周期内的最新发布的版本，下拉条件以版本发布时间“从新至旧”显示；版本对比下拉复选最多选择4个版本号；当选中超过4个时不可选，并在筛选器后方提示“最多选择4个版本对比”；当选择时间内，无版本号时，在筛选器后方提示“该周期下无版本，请重新选择时间”；
    • 4.图标类型：包含“版本新增用户”、“版本活跃用户”、“版本注册用户”、“版本付费用户”、“版本留存用户”；各用户指标定义：
      • 4.1新增用户：所选周期范围内，安装应用后首次启用应用的用户；
      • 4.2活跃用户：所选周期范围内（去重），至少启动过一次的用户；
      • 4.3注册用户：所选周期范围内，有注册成功行为的用户；
      • 4.4付费用户：所选周期范围内，有付费成功行为的用户；
      • 4.5留存用户：日留存用户=第一天新增用户在第二天登录过的用户，以日存储次日留存用户； 周留存用户=第一周新增用户在第二周登录过的用户；月留存用户=第一月新增用户在第二月登录过的用户；留存用户数：日留存>2天，周留存>2周，月留存>2月；
    • 5.单位规则：X轴日期文本最多显示15条数据，超出的数据平均分布显示，当鼠标移入图表后tips显示详情； 图表的y纵轴坐标中单位处理包含：
      • 5.1当为客流时：1万人以下按人，1万人及以上按1万人，10千万以下按千万人，10千万人及以上按1亿人；
      • 5.2全部客流y轴0后面增加“/单位”，单位按照y纵轴最大值单位显示，当小于最大单位数值时，显示零点几数值，客流零点小数点后保留1位；
      • 5.3数值格式：每三位用“，”逗号隔开;
    • 3.4画像分析
    • 在“画像分析”里，以全部用户类型、多维度、多事件交叉分析，为解决问题提供15组标签，包含设备品牌、操作系统、屏幕分辨率、网络方式、运营商、年龄、性别、婚姻、教育、省份、城市、职业、月收入、兴趣偏好、APP偏好数据；
    • 画像分析指标说明：
    • 1. 可通过不同用户类型&不同维度、不同事件、不同周期进行多维交叉分析；
    • 2. 用户类型：以“新增用户、活跃用户、注册用户、付费用户”类型筛选；
    • 3. 维度：各个渠道&各个活动；
    • 4. 事件：各个事件组；
    • 5. 时间维度：以天“T+1自定义选择”，以月、季度、年，自定义选择；
    • 6. 标签分类为：
      • 6.1设备品牌，6.2 操作系统，6.3 屏幕分辨率，6.4 网络方式，6.5 运营商，6.6 年龄，6.7 性别，6.8 婚姻，6.9 教育，6.10 来源/省份/城市，6.11 职业，6.12 月收入，6.13 兴趣偏好，6.14 APP偏好
  • 4.权限管理
    • 通过“权限管理”模块实现自配置用户、自配置角色，配置不同用户可以访问不同动作模块；
    • 4.1用户管理
    • 超级管理员可以在产品中分配用户，便于不同的用户访问使用；
    • 用户管理指标说明：
    • 1. 新增用户：用户信息：用户名、邮箱、状态、手机号、添加角色（选择角色、搜索角色）；
    • 2. 查看用户：同新建用户字段、可编辑用户；
    • 3. 编辑用户：同新建用户字段；
    • 4. 删除用户：删除所选用户后，用户将不可访问平台；
    • 5. 搜索用户：当搜索结果为空时，提示：暂无搜索记录；当搜索有数据时，高亮显示搜索关键词；
    • 4.2角色管理
    • 超级管理员可以在产品中分配角色，便于不同的角色使用不同的功能模块；
    • 角色管理指标说明：
    • 1. 新增角色：角色信息：角色名称、添加用户（选择用户、搜索用户），分配模块&动作；
    • 2. 查看角色：同新建角色字段、可编辑角色；
    • 3. 编辑角色：同新建角色字段；
    • 4. 删除角色：删除所选角色后，拥有该角色的用户将失去该角色的权限关联关系；
    • 5. 搜索角色：当搜索结果为空时，提示：暂无搜索记录；当搜索有数据时，高亮显示搜索关键词；
